各種申請管理システム (Client-Server Application System)
C言語で開発した、クライアント・サーバ型の汎用申請管理システムです。
ソケット通信とマルチスレッド(pthread)による並行処理を実装し、メタデータ（検索用）をPostgreSQL、可変長の文字列データ（申請内容）をテキストファイルとしてローカルディレクトリに隔離保存する「完全対称ハイブリッド・アーキテクチャ」を採用しています。

特徴
独自プロトコル通信: 生のTCP/IPソケットAPIを利用し、APPADD, FMTGET などの独自コマンドベースで通信を行います。

マルチスレッド処理: pthread を用いて、複数クライアントからの同時接続を安全に処理します。

ハイブリッド・データ管理:

メタデータ (ID、提出者、ステータス、タイムスタンプ) は PostgreSQL (libpq) で高速に一元管理。

動的で可変長なユーザー入力データや様式の設計図はテキストファイルとして保存し、DBの肥大化とSQLインジェクションを構造的に防ぎます。

階層型ロールベースのアクセス制御 (RBAC): admin, teacher, student の3層の権限を持ち、上位権限は下位権限を内包する設計です。

動的フォーマットシステム: データベースのテーブル構造(ALTER TABLE)を変更することなく、無限に新しい様式（入力項目）を追加・運用できます。

環境要件
Debian/Ubuntu (WSL2環境等) を想定しています。以下のパッケージが必要です。

GCC (Cコンパイラ)

PostgreSQL (サーバー本体)

libpq-dev (C言語用 PostgreSQL クライアントライブラリ)

Bash

# Ubuntu / Debian系でのインストール例

sudo apt update
sudo apt install gcc postgresql postgresql-contrib libpq-dev
📁 ディレクトリ構成 (Directory Structure)
初回起動前に、可変長データを保存するための専用ディレクトリを作成する必要があります。

Plaintext
.
├── client.c, client.h, client_*.c  # クライアント側ソースコード
├── server.c, server.h, controller.c, app*.c, format*.c, user*.c # サーバー側ソースコード
├── apps_data/   # (必須) 申請書の回答データが保存されるディレクトリ
└── formats/     # (必須) 様式の設計図データが保存されるディレクトリ
⚙️ セットアップ (Setup)

1. データベースの準備
PostgreSQLにシステム用のデータベースとテーブルを作成します。（※テーブル定義用のSQLファイルがある場合は適宜実行してください）

2. データ保存用ディレクトリの作成
サーバーのルートディレクトリに、データを格納するディレクトリを作成します。

Bash
mkdir apps_data
mkdir formats
3. コンパイル
Bash
gcc -o server *.c -lpthread -lpq
gcc -o client*.c
※コンパイルが成功すると、server および client の実行ファイルが生成されます。

使い方 (Usage)
サーバーの起動
別のターミナル（またはバックグラウンド）でサーバーを起動して、クライアントからの接続を待機させます。

Bash
./server
クライアントの起動
クライアントプログラムを実行します。

Bash
./client
主な機能 (メニュー)
ゲスト状態からログインすることで、権限に応じた以下の機能にアクセスできます。

【様式管理】: 様式一覧取得、新規様式作成、様式更新

【申請管理】: 申請一覧取得、詳細確認、新規申請、修正、取り下げ、申請書ダウンロード (LuaLaTeX/CSV)

【承認・通知】: 新着通知確認、申請の承認・差し戻し

【ユーザー管理】: ユーザー登録、情報更新、削除
